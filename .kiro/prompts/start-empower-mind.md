# Start Empower Mind

Use the `web-orchestrator` agent in Kiro or follow root `AGENTS.md` in Codex.

Read all global Sebastian Web Core steering and every workspace file under `.kiro/steering/`, `.kiro/specs/empower-mind-website/`, and `project-content/`. Inspect the Astro repository before editing.

This repository is a static Astro foundation for Empower Mind and is configured for Cloudflare Pages under project name `empower-mind`. Preserve pnpm, Astro static output, `dist`, semantic HTML, custom CSS, and Pages compatibility.

Proceed in this order:

1. Report what exists and any conflicts with workspace steering.
2. Classify organization content as confirmed, provisional, missing, or retired.
3. Review the route architecture, content model, typography, layout, motion, responsive behavior, accessibility, and Cloudflare Pages configuration.
4. Improve only the first coherent vertical slice: header, opening, context, and response model.
5. Do not fabricate programs, metrics, stories, partners, services, crisis routes, or contact details.
6. Run `pnpm check`, `pnpm lint`, `pnpm test`, and `pnpm build`.
7. Stop with a precise report and next milestone.

Do not generate the entire organization website in one pass. The current routes are structural placeholders until approved content is supplied.
